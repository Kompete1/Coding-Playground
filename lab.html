How this lab works

A simple loop I use to practise GitHub, Windsurf, Codex and the OpenAI platform with small, safe changes.

  1. 1. Pick a tiny experiment

    Choose one small change: a visual tweak, a new card, a widget, or a behaviour to test. Keep the scope small enough to finish in one session.

  2. 2. Create or select a feature branch

    Name the branch clearly (for example, feat/home-loan-calculator or ui/faq-readability) so it‚Äôs obvious what the change is about.

  3. 3. Ask Codex to help implement

    Use Codex in the cloud or in Windsurf to plan the change, see the file diffs, and only then approve updates to the branch.

  4. 4. Open a draft pull request

    On GitHub, open a draft PR so you can review the diff, add notes, and connect the code change to a specific experiment card in this lab.

  5. 5. Preview locally in Windsurf

    Pull the branch into Windsurf, run the local preview (for example http://localhost:8000), and click through the demo link from the experiment card.

  6. 6. Merge, deploy, then update the card

    Once you‚Äôre happy, merge the PR, pull main into Windsurf, confirm it‚Äôs live on GitHub Pages, and then update or create the experiment card here.
